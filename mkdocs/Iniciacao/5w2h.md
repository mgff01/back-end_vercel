## Introdução

O 5W2H é uma ferramenta de gestão estratégica amplamente utilizada para o planejamento e a execução de ações. Baseada em sete perguntas fundamentais (What?, Why?, Where?, Who?, When?, How? e How much?), ela tem o intuito de esclarecer e estruturar ideias de forma objetiva. Neste documento, o 5W2H é aplicado para definir as diretrizes do Sistema de Validação de Estágio, servindo como um guia prático para alinhar os objetivos, responsabilidades e a tomada de decisão da equipe durante o desenvolvimento do projeto.

## Metodologia

A equipe se reuniu para esclarecimento sobre o tema do projeto, a partir dos pontos debatidos foi possível responder às questões do 5W2H.

## Questões 5W2H

### Visão produto

| **Questão**                     | **Resposta**                                                         |
| ------------------------------- | -------------------------------------------------------------------- |
| What? (O que deve ser feito?)   | Sistema de validação de estágio integrado ao processo administrativo |
| Why? (Por que isso será feito?) | Para auxiliar na integração e otimizar o processo de estágio         |
| Where? (Onde será feito?)       | No Ibmec                                                             |
| When? (Quando será feito?)      | Durante o semestre vigente                                           |
| Who? (Quem irá fazer?)          | Alunos da disciplina Projeto Back-end                                |
| How? (Como será feito?)         | Utilizando metodologia RUP/UP e ferramentas de desenvolvimento web   |
| How much? (Quanto custará?)     | Tempo e esforço dedicado pela equipe no desenvolvimento              |

## Conclusão

A elaboração do 5W2H permitiu à equipe consolidar a visão geral do Sistema de Validação de Estágio de forma rápida e objetiva. Ao responder as sete perguntas, ficaram evidentes as motivações, o escopo de atuação e as responsabilidades da equipe. Com esse alinhamento inicial estabelecido, nós possuímos uma base sólida para avançar com clareza nas próximas etapas, o brainstorm, a elicitação de requisitos e desenvolvimento da aplicação.
